8 /* Note to developers:
9  *
10  * Why did the authors create structs that contain only an array?
11  * Wouldn't it be easier to simply refer to the array instead of
12  * unwrapping it from the context of the struct each time it is
13  * invoked? You could even use a typedef to further shorten each invocation!
14  *
15  * Wisdom, thrown down from the mountain:
16  * http://stackoverflow.com/a/4523537 [quoted below]
17  *
18  * [An unwrapped array with a typedef for convenience] is probably a
19  * very bad idea, because the resulting type is an array type, but
20  * users of it won't see that it's an array type. If used as a
21  * function argument, it will be passed by reference, not by value,
22  * and the sizeof for it will then be wrong.
23  */
